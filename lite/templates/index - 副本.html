02. Our solutions Data Integration for everyone

Out-of-the-box Integrator supports numerous highly configurable data integration scenarios.

Data Export

Export data from relational databases, cloud data sources, public and private APIs and social networks. If your database or service is behind the firewall, we can install a small, zero-maintenance integration agent which will push data to the cloud.

Data Import

Load data from files, relational databases, cloud sources, public and private APIs, social networks to other databases, cloud sources and APIs. You can map source and destination fields and create very complex transformations.

Data Synchronization

Set up a real-time data synchronization between different data sources with minimum effort. Out-of-the-box Integrator supports data synchronization for Oracle, SQL Server, MySql, PostgreSQL, DB2, Informix and Sybase as well as any JDBC compatible database and web services

Backup

Create a copy of the relational database or cloud data in another database or file storage, such as Amazon S3, Google Cloud Storage, FTP and SFTP. For example, you can point Integrator to the API endpoint which returns a complex nested data in JSON format and Integrator will automatically create and load data into the multiple staging tables.

Automation

Build very complex, event-driven workflow with multiple integration points and decision branches. Use a wizard-based drag and drop interface as well as JavaScript and SQL to create data integration scenarios.

Connect to APIs and build APIs

Connect to practically any REST and SOAP API, as well as APIs based on industry standard EDI formats, such as HL7. Build complex APIs with just a few mouse clicks.

What is Integrator?

Integrator is a cloud data integration service which can also be installed on premise.

How Integrator is different from other similar services and products?

Integrator is a powerful and easy-to-use cloud data integration service which can work with data behind the corporate firewall. Users can create very complex data integration scenarios in no time, right in the browser, using intuitive drag and drop interface. Also, Integrator is a full blown Enterprise Service Bus, so users can easily build data integration APIs. Last but not least, Integrator supports unique data integration scenarios, typically requiring a specialized set of tools and costing a fortune to implement, such as: data synchronization, HL7 transformations, direct data upload into the online data-warehouses such as Snowflake, and many others.

Is it possible to add new functionality to the Integrator?

Integrator has been designed to be expandable. Connectors, data integration modules, dashboards and other components are implemented as plugin modules and can be added or removed on the fly without restarting the service.

Can we install Integrator on premise?

Yes you can. Read more about our on premise options.

Installing Integrator on premise is not an option. Can it still work with the data behind the firewall?

No worries, we have got you covered. Please read below.

My database is in a corporate network, behind the firewall. How can Integrator access it from the cloud?

We install and configure a zero-maintenance background service called Integration Agent right in your network, which talks to your corporate database and to the Integrator on the cloud.

How does the Integration Agent communicate with a databases and APIs in a corporate network?

Integration Agent works behind the corporate firewall, so it communicates with databases and APIs directly. It uses JDBC to talk to the databases and HTTP to get and push data to REST and SOAP APIs.

How does the Integration Agent, working behind the corporate firewall, communicate with Integrator on a cloud?

Integration Agent can push data to the secure cloud file storage such as Amazon S3, which is accessible by Integrator, or it can push data directly to the Integrator REST API.

Is it possible to load data into my corporate database or API using the Integration Agent?

Absolutely. Our Integration Agent can load data into the database or API just as well as extract it.

So, why do I need a cloud-based Integrator if I have the Integration Agent working behind my corporate firewall?

Integration Agent is limited to simple change replication, extract and load scenarios. Integrator and Integration Agent work together to support very complex data integration flows with transformations, mapping and logic.

Can Integrator connect to to my database/application/service/etc?

Almost certainly - it can. Please see the list of the available connectors. If connector you are looking for is not on a list - we or our partners can develop it practically in no time.

Can Integrator connect to the web services?

Integrator supports REST and SOAP based web services through an HTTP connector. You specify the endpoint, type of the payload and format (JSON, XML, etc.), as well as any authentication parameters and Integrator will handle the rest. Integrator will automatically connect to the endpoint, push the payload and parse the response. There is no difference between working with the web services compare to working with the files downloaded locally.

Can Integrator connect to the well know APIs?

Integrator includes a growing list of connectors to the well knows APIs, such as Google Analytics, Facebook, Twitter, Marketo, etc. We are constantly adding new connectors so please stay tuned.

Does integrator support any of the standard EDI formats such as HL7?

Integrator fully supports HL7 v 2.x and 3.x. We are working on adding connectors to other formats. Please stay tuned.

Does Integrator have its own REST APIs so we can integrate our system with it?

In Integrator you can configure API endpoints, which will be listening to the incoming HTTP requests and, once payload is received, will trigger an execution of the data integration flows. You can also expose data sets, available from the Integrator, as REST API endpoints, build CRUD APIs and execute flows, created in Integrator, using REST API. Read more about integrations and APIs and how to build REST APIs in Integrator.

Can we trust our data to you?

We don’t have access to your data at all unless you opt-in to store it on our servers. If needed, we can install Integrator on any hardware managed by you, including cloud providers such as Amazon and Microsoft.

How do you protect data?

We take security very seriously. We only use SSL connections, all our API endpoints are protected by short-lived JWT tokens and we encrypt all passwords, security keys and tokens.

How do you encrypt credentials?

In Integrator all passwords, access keys, and other secure credentials are encrypted by strong encryption algorithm with 512-bit key.
